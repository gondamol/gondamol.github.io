🚀 Quick Deployment Guide

Option 1: Automated Script (Recommended)

./deploy.sh

This script will: 1. ✅ Check for changes 2. ✅ Show what will be committed
3. ✅ Ask for commit message 4. ✅ Stage only source files (ignores _site/) 5. ✅ Commit changes 6. ✅ Push to GitHub main branch 7. ✅ Ask if you want to publish to GitHub Pages

Option 2: Manual Deployment

Step 1: Stage Source Files Only

git add index.qmd
git add about/index.qmd
git add cv/index.qmd
git add skills/index.qmd
git add _quarto.yml
# Add other modified .qmd files as needed

Step 2: Commit with Good Message

git commit -m "Brief description of changes

- Key change 1
- Key change 2
- Key change 3"

Step 3: Push to GitHub

git push origin main

Step 4: Publish to GitHub Pages

quarto publish gh-pages

📝 Commit Message Best Practices

Good Examples:

git commit -m "Add new project showcase for KEMRI cancer dashboard"

git commit -m "Update CV with recent publications and certifications"

git commit -m "Enhance homepage hero section and statistics"

git commit -m "Fix typos in about page and update contact information"

Bad Examples:

git commit -m "update"
git commit -m "changes"
git commit -m "fix"

🔍 Useful Git Commands

Check what changed:

git status
git diff

See commit history:

git log --oneline -10

Undo last commit (keep changes):

git reset --soft HEAD~1

Discard uncommitted changes:

git restore <filename>

🎯 Files to ALWAYS Commit

✅ Source Files (.qmd) - index.qmd - about/index.qmd - cv/index.qmd - skills/index.qmd - projects/*.qmd

✅ Configuration - _quarto.yml - styles.scss - custom.scss

✅ Documentation - README.md - Any .md files

❌ Files to NEVER Commit

❌ Generated Files - _site/* (auto-deleted from main branch) - .quarto/* (already in .gitignore)

❌ Temporary Files - .Rproj.user - .Rhistory - .RData

🌐 After Publishing

Your site updates live at: https://gondamol.github.io

⏰ Wait 1-2 minutes for GitHub Pages to rebuild

🔄 Force refresh your browser: Ctrl+Shift+R (Windows/Linux) or Cmd+Shift+R (Mac)

🆘 Troubleshooting

Problem: “_site/ files showing in git status”

git rm -r --cached _site/
git commit -m "Remove _site/ from git tracking"
git push origin main

Problem: “Merge conflict”

# Pull latest changes first
git pull origin main

# Resolve conflicts in your editor
# Then commit and push
git add .
git commit -m "Resolve merge conflicts"
git push origin main

Problem: “Site not updating on GitHub Pages”

  • Wait 2-3 minutes
  • Check GitHub Actions tab in your repo
  • Try force refresh in browser
  • Run quarto publish gh-pages again

Pro Tip: Run ./deploy.sh after every significant change to keep your live site up-to-date! 🎉
